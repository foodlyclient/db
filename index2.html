Documentation - HMAC-SHA256

Qu'est-ce que HMAC-SHA256 ?

HMAC-SHA256 (Hash-based Message Authentication Code avec SHA-256) est un algorithme cryptographique qui combine une fonction de hachage (SHA-256) avec une clé secrète pour produire une signature numérique. Cette signature permet de :

  • Vérifier l'intégrité : S'assurer que le message n'a pas été modifié
  • Vérifier l'authenticité : Confirmer que le message vient bien de l'expéditeur légitime
  • Prévenir les attaques : Les signatures HMAC sont résistantes aux attaques par collision

Comment ca marche ?

Le processus HMAC utilise deux passes de hachage :

  1. Preparation des clés : La clé secrète est étendue ou réduite à la taille appropriée
  2. Premiere passe : Hachage avec une clé XORée avec ipad (0x36)
  3. Deuxieme passe : Hachage du resultat precedent avec une clé XORée avec opad (0x5C)
  4. Resultat : Signature de 256 bits (64 caracteres hexadecimaux)
Formule HMAC :
HMAC(K, M) = H((K ⊕ opad) || H((K ⊕ ipad) || M))

Utilisation de l'outil

1. Generer une signature

  1. Allez dans l'onglet "Generer Signature"
  2. Entrez votre message dans le champ "Message a signer"
  3. Entrez votre cle secrete dans le champ "Cle secrete"
  4. Cliquez sur "Generer Signature"
  5. Copiez ou telechargez la signature obtenue

2. Verifier une signature

  1. Allez dans l'onglet "Verifier Signature"
  2. Entrez le message original
  3. Entrez la meme cle secrete utilisee pour generer
  4. Collez la signature a verifier (format hex)
  5. Cliquez sur "Verifier Signature"

3. Generer le code C

  1. Assurez-vous d'avoir entre une cle dans les onglets precedents
  2. Allez dans l'onglet "Code C"
  3. Cliquez sur "Injecter la Cle" pour inserer votre cle dans le code
  4. Telechargez le fichier "verificateur.c"
  5. Compilez avec : gcc -o verificateur verificateur.c -lssl -lcrypto

Compilation et utilisation du code C

Prérequis système

Avant de compiler le code C généré, assurez-vous d'avoir installé les bibliothèques nécessaires :

  • Ubuntu/Debian : sudo apt-get install build-essential libssl-dev
  • CentOS/RHEL : sudo yum install gcc openssl-devel
  • macOS : brew install openssl (avec Homebrew)
  • Windows : Installez MinGW-w64 ou MSYS2 avec OpenSSL

Procédure de compilation

Suivez ces étapes pour compiler le vérificateur HMAC-SHA256 :

Étape 1 : Téléchargez le code C
# Dans l'onglet "Code C" de l'application
Cliquez sur "Injecter la Cle"
Cliquez sur "Telecharger Code C"

Étape 2 : Compilation avec GCC
gcc -o verificateur verificateur.c -lssl -lcrypto

Étape 3 : Vérification de la compilation
ls -la verificateur
# Doit afficher un fichier exécutable

Utilisation du programme compilé

Le programme compilé peut être utilisé de deux manières :

Mode interactif
./verificateur
# Le programme vous demandera le message et la signature
Mode arguments (recommandé pour l'automatisation)
./verificateur "votre_message" "signature_hex"

Exemple concret :
./verificateur "Hello World" "a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e49"
# Retourne : Signature valide

Dépannage courant

Erreur : "openssl/ssl.h: No such file or directory"
# Solution : Installer les headers OpenSSL
sudo apt-get install libssl-dev

Erreur : "undefined reference to 'HMAC'"
# Solution : Vérifier les flags de compilation
gcc -o verificateur verificateur.c -lssl -lcrypto -I/usr/include/openssl

Erreur : "permission denied"
# Solution : Rendre le programme exécutable
chmod +x verificateur

Options de compilation avancées

Pour optimiser les performances en production :

gcc -O3 -march=native -o verificateur verificateur.c -lssl -lcrypto
# Optimisation niveau 3 + optimisations CPU spécifiques

gcc -g -Wall -Wextra -o verificateur verificateur.c -lssl -lcrypto
# Compilation avec debug et warnings étendus

Bonnes pratiques de securite

  • Cles longues : Utilisez des cles d'au moins 256 bits (32 caracteres)
  • Cles uniques : Une cle differente par usage/service
  • Rotation : Changez regulierement vos cles secretes
  • Stockage securise : Ne stockez jamais les cles en clair dans le code
  • HTTPS : Utilisez toujours HTTPS pour transmettre les signatures
  • Timing attacks : Comparez les signatures de maniere constante en temps

Implementation technique

JavaScript (Web Crypto API)

const key = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(secretKey),
    { name: 'HMAC', hash: 'SHA-256' },
    false,
    ['sign']
);
const signature = await crypto.subtle.sign('HMAC', key, messageBuffer);

C (OpenSSL)

HMAC_CTX *ctx = HMAC_CTX_new();
HMAC_Init_ex(ctx, key, key_len, EVP_sha256(), NULL);
HMAC_Update(ctx, message, strlen(message));
HMAC_Final(ctx, result, &len);
HMAC_CTX_free(ctx);

Compatibilite et formats

  • Format de sortie : Hexadecimal (64 caracteres pour SHA-256)
  • Encodage : UTF-8 pour les messages texte
  • Cross-platform : Compatible entre JS et C avec les memes parametres
  • Longueur fixe : Toutes les signatures font exactement 64 caracteres hex
